# Bina Type System

## Общие принципы

Система типов языка Bina основана на следующих правилах:

- **Номинальная типизация** — два типа совместимы тогда и только тогда,
  когда их имена совпадают. Структурная эквивалентность не используется.
- **Value semantics** — все значения передаются и присваиваются по значению (копирование).
  Язык не имеет ссылочных типов, указателей или заимствований.
- **Обязательная инициализация** — каждая переменная должна быть инициализирована
  в момент объявления. Использование неинициализированной переменной —
  ошибка компиляции.

## Встроенные типы

### Целочисленные типы

| Тип | Размер | Диапазон |
|-----|--------|----------|
| `int8` | 8 бит | −128 .. 127 |
| `int16` | 16 бит | −32 768 .. 32 767 |
| `int32` | 32 бит | −2 147 483 648 .. 2 147 483 647 |
| `int64` | 64 бит | −2⁶³ .. 2⁶³−1 |
| `uint8` | 8 бит | 0 .. 255 |
| `uint16` | 16 бит | 0 .. 65 535 |
| `uint32` | 32 бит | 0 .. 4 294 967 295 |
| `uint64` | 64 бит | 0 .. 2⁶⁴−1 |

`int` — псевдоним для `int64`. Используется как тип возврата функции `main`.

### Вещественные типы

| Тип | Размер |
|-----|--------|
| `float32` | 32 бит |
| `float64` | 64 бит |

### Прочие типы

| Тип | Описание |
|-----|----------|
| `bool` | Булевое значение: `true` или `false` |
| `string` | Строка в кодировке UTF-8, неизменяема |
| `void` | Отсутствие значения. Допускается только как тип возврата функции |

## Составные типы

### Массивы

```
[N]T
```

Массив фиксированного размера `N` элементов типа `T`.
Размер массива является частью типа: `[5]int32` и `[6]int32` — разные, несовместимые типы.

Пример:
```bina
let arr: [3]float64 = [1.0, 2.0, 3.0];
```

Количество элементов в литерале массива должно точно совпадать с размером типа.

### Структуры

Именованный составной тип с полями. Каждое поле имеет имя и тип.

```bina
struct Point {
    x: float64;
    y: float64;
}
```

Два типа структур совместимы только если это одна и та же структура (номинальная типизация).
Структура `A` с полями `{ x: int32 }` и структура `B` с полями `{ x: int32 }` — разные типы.

---

## Синонимы типов

Синоним типа вводит альтернативное имя для существующего типа.

```bina
type Celsius    = float64;
type Fahrenheit = float64;
```

Переменная типа `Celsius` может быть присвоена переменной типа `Fahrenheit`,
и наоборот, так как это один и тот же тип `float64`.

## Типы литералов

### С явным суффиксом

Если литерал имеет суффикс, его тип определяется суффиксом:

| Литерал | Тип |
|---------|-----|
| `42i8` | `int8` |
| `42i16` | `int16` |
| `42i32` | `int32` |
| `42i64` | `int64` |
| `42u8` | `uint8` |
| `42u16` | `uint16` |
| `42u32` | `uint32` |
| `42u64` | `uint64` |
| `3.14f32` | `float32` |
| `3.14f64` | `float64` |

### Без суффикса (вывод типа по умолчанию)

| Литерал | Тип по умолчанию |
|---------|------------------|
| `42` | `int32` |
| `3.14` | `float64` |
| `"hello"` | `string` |
| `true`, `false` | `bool` |

## Вывод типов

При объявлении переменной указание типа необязательно.
Если тип опущен, компилятор выводит его из выражения-инициализатора.

```bina
let x = 42;          // int32
let y = 3.14;        // float64
let s = "hello";     // string
let f = true;        // bool
let b = 255u8;       // uint8
```

Правила вывода:

| Инициализатор | Выведенный тип |
|---------------|----------------|
| Целочисленный литерал без суффикса | `int32` |
| Целочисленный литерал с суффиксом | тип суффикса |
| Вещественный литерал без суффикса | `float64` |
| Вещественный литерал с суффиксом | тип суффикса |
| Строковый литерал | `string` |
| Булевый литерал | `bool` |
| Вызов функции | тип возврата функции |
| Литерал массива | `[N]T`, где `N` — количество элементов, `T` — тип элементов |
| Литерал структуры | тип указанной структуры |
| Переменная | тип переменной |
| Бинарное/унарное выражение | тип результата операции |

Если компилятор не может однозначно определить тип — ошибка компиляции.

~~Вывод типа возвращаемого значения функции не поддерживается —
тип возврата функции всегда указывается явно.~~

## Неявное приведение типов

В арифметических и сравнительных операциях допускается автоматическое расширение числовых типов. Сужение запрещено — требуется явный `cast`.

### Иерархия расширения

```
int8 → int16 → int32 → int64
         ↓        ↓        ↓
uint8 → uint16 → uint32 → uint64

int32 → float32
int32 → float64
int64 → float64

float32 → float64
```

### Правила

1. Если оба операнда имеют одинаковый тип — результат имеет этот тип.
2. Если один операнд может быть расширен до типа другого без потери данных —
   компилятор автоматически выполняет расширяющее приведение.
3. Знаковые и беззнаковые типы одного размера не совместимы неявно —
   требуется явный `cast`.
4. Строки, bool и void не участвуют в неявном приведении.

### Примеры

```bina
let a: int32 = 10;
let b: float64 = 3.14;
let c = a + b;             // a неявно расширяется до float64, результат: float64

let x: int8 = 100;
let y: int32 = x + 1;      // x неявно расширяется до int32

let z: int8 = x + 1;       // результат int32 не может быть сужен до int8
let z: int8 = cast<int8>(x + 1);  // явное приведение
```

### Приведение при присваивании

При присваивании и инициализации неявное расширение также допускается:

```bina
let big: int64 = 42i32;    // int32 → int64 (расширение)
let small: int8 = 42i32;   // int32 → int8 (сужение, требуется cast)
```

## Явное приведение типов

Оператор `cast<T>(expr)` выполняет явное преобразование значения к типу `T`.

Допустимые преобразования:

| Из | В | Поведение |
|----|---|-----------|
| Целое | Целое | Обрезание старших бит при сужении |
| Целое | Вещественное | Преобразование с возможной потерей точности |
| Вещественное | Целое | Отбрасывание дробной части (truncation) |
| Вещественное | Вещественное | Округление при сужении |
| `bool` | Целое | `false` → 0, `true` → 1 |
| Целое | `bool` | 0 → `false`, ≠ 0 → `true` |

Недопустимые преобразования (ошибка компиляции):

- `string` ↔ числовой тип
- `string` ↔ `bool`
- Массив ↔ что-либо
- Структура ↔ что-либо

---

## Поведение при переполнении (Overflow)

Переполнение целочисленных типов — ошибка во время выполнения.
При возникновении переполнения программа завершается с сообщением:

```
runtime error: integer overflow at line <N>
```

Переполнение вещественных типов приводит к значениям `±Inf` или `NaN`.
