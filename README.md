# bina-compiler

Компилятор для языка **Bina** — статически типизированного императивного языка с явной мутабельностью. Реализован на C++23.

## Сборка

```bash
cmake --preset default
cmake --build build
```

Требуется: `clang++-22`, `clang-scan-deps-22`, `libc++-22`, `llc-22`,
C-компилятор (`cc`), CMake ≥ 3.30, Ninja.

## Запуск

```bash
./build/bina <source.bina> -o <output>  # сборка исполняемого файла
./build/bina --emit-ir <source>         # вывод LLVM IR в stdout
./build/bina --no-link <source> -o obj  # сборка object-файла без линковки
./build/bina --dump-tokens <source>     # вывод потока токенов
./build/bina --dump-ast <source>        # вывод AST
./build/bina --dump-symbols <source>    # вывод глобальной области видимости
```

Запуск без dump-флагов выполняет полный pipeline: lexer → parser → semantic →
LLVM IR → object-файл → executable. Runtime-объект `bina_runtime.o` копируется
рядом с `build/bina` при сборке.

Примеры программ — в [examples/](examples/).

## Дополнительные возможности

Нумерация заданий — по ТЗ v2.0 ([projectv2.0.md](projectv2.0.md)).

- **[A.1.1](projectv2.0.md#L514) Числовые типы разных размеров** — поддерживаются `int8`...`int64`, `uint8`...`uint64`, `float32`/`float64`, числовые суффиксы (`42i32`, `255u8`, `3.14f32`), hex/binary-литералы (`0xFF`, `0b1010`), экспоненциальная запись float и специальные значения `inf`/`nan`/`NaN`.
- **[A.1.2](projectv2.0.md#L516) Unicode-строки и символы (UTF-8)** — поддержаны типы `string` и `char`: строки UTF-8 с escape-последовательностями, конкатенация, сравнение и `len` (длина в байтах); `char` хранит 32-битный Unicode scalar value, поддерживает символьные литералы, `code` и `char_from`.
- **[A.1.7](projectv2.0.md#L526) Вывод типов** — частичная реализация: тип переменной `let`/`mut` может выводиться по инициализатору (`let x = 42;`); вывод типа возвращаемого значения функций не поддерживается.
- **[A.1.8](projectv2.0.md#L528) Неявное приведение типов** — автоматическое расширение числовых типов в выражениях, инициализации, присваивании, `return` и вызовах функций; таблица преобразований — в [specs/types.md](specs/types.md).
- **[A.2.12](projectv2.0.md#L590) Методы структур (`impl`)** — поддержка методов через блоки `impl`, вызов через точку (`obj.method()`) с автоподстановкой `self`, а также явный вызов `Struct::method(obj)`. Вторая часть задания — видимость членов (`pub`/private) — в работе.
- **[A.2.8](projectv2.0.md#L580) Перегрузка функций и методов** — несколько функций или методов с одним именем, различающихся сигнатурой.
- **[A.3.1](projectv2.0.md#L622) Разрешение перегрузок с неявными приведениями** — при выборе перегрузки учитываются допустимые числовые расширения; неоднозначные вызовы диагностируются. В работе: уровни соответствия EXACT/PROMOTION/CONVERSION и выбор более конкретного кандидата.
- **[B.1.1](projectv2.0.md#L678) Восстановление после синтаксических ошибок** — парсер синхронизируется после ошибки и может вывести несколько диагностик за один запуск.
- **[B.1.8](projectv2.0.md#L698) Дополнительные режимы CLI** — поддержаны диагностические и backend-флаги: `--dump-tokens`, `--dump-ast`, `--dump-symbols`, `--emit-ir`, `--no-link` и `-o`.

Полный список выбранных заданий, статусы и расчёт баллов — в [dops.md](dops.md),
план реализации оставшихся — в [impl_plan.md](impl_plan.md).
