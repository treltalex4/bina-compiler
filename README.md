# bina-compiler

Компилятор для языка **Bina** — статически типизированного императивного языка с явной мутабельностью. Реализован на C++23.

## Сборка

```bash
cmake --preset default
cmake --build build
```

Требуется: `clang++-22`, `clang-scan-deps-22`, `libc++-22`, CMake ≥ 3.30, Ninja.

## Запуск

```bash
./build/bina <source.bina>              # frontend-проверка: lexer + parser + semantic
./build/bina --dump-tokens <source>     # вывод потока токенов
./build/bina --dump-ast <source>        # вывод AST
./build/bina --dump-symbols <source>    # вывод глобальной области видимости
```

Генератор кода пока не реализован, поэтому запуск без dump-флагов выполняет
проверку программы до семантического анализа включительно.

Примеры программ — в [examples/](examples/).

## Дополнительные возможности

- **[A.1.1](projectv1.0.md#L483) Числовые типы разных размеров** — поддерживаются `int8`...`int64`, `uint8`...`uint64`, `float32`/`float64`, числовые суффиксы (`42i32`, `255u8`, `3.14f32`), hex/binary-литералы (`0xFF`, `0b1010`), экспоненциальная запись float и специальные значения `inf`/`nan`/`NaN`.
- **[A.1.2](projectv1.0.md#L485) UTF-8 строки и символы** — поддержаны типы `string` и `char`: строки UTF-8 с escape-последовательностями, конкатенация, сравнение и `len`; `char` хранит 32-битный Unicode scalar value, поддерживает символьные литералы, `code` и `char_from`.
- **[A.1.6](projectv1.0.md#L493) Вывод типов** — частичная реализация: тип переменной `let`/`mut` может выводиться по инициализатору (`let x = 42;`); вывод типа возвращаемого значения функций не поддерживается.
- **[A.1.7](projectv1.0.md#L495) Неявное приведение типов** — автоматическое расширение числовых типов в выражениях, инициализации, присваивании, `return` и вызовах функций.
- **[A.2.3](projectv1.0.md#L519) Методы структур (`impl`)** — поддержка методов через блоки `impl`, вызов через точку (`obj.method()`) с автоподстановкой `self`, а также явный вызов `Struct::method(obj)`.
- **[A.2.9](projectv1.0.md#L531) Перегрузка функций и методов** — несколько функций или методов с одним именем, различающихся сигнатурой.
- **[A.3.1](projectv1.0.md#L549) Разрешение перегрузок с неявными приведениями** — при выборе перегрузки учитываются допустимые числовые расширения; неоднозначные вызовы диагностируются.
- **[B.1.1](projectv1.0.md#L577) Восстановление после синтаксических ошибок** — парсер синхронизируется после ошибки и может вывести несколько диагностик за один запуск.
- **[B.1.5](projectv1.0.md#L585) Дополнительный CLI-режим** — частичная реализация: флаг `--dump-symbols` выводит собранную таблицу символов.
