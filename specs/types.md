# Bina Type System

## Общие принципы

Система типов языка Bina основана на следующих правилах:

- **Номинальная типизация** — два типа совместимы тогда и только тогда,
  когда их имена совпадают. Структурная эквивалентность не используется.
- **Value semantics** — все значения передаются и присваиваются по значению (копирование).
  Язык не имеет ссылочных типов, указателей или заимствований.
- **Обязательная инициализация** — каждая переменная должна быть инициализирована
  в момент объявления. Использование неинициализированной переменной —
  ошибка компиляции.

## Встроенные типы

### Целочисленные типы

| Тип | Размер | Диапазон |
|-----|--------|----------|
| `int8` | 8 бит | −128 .. 127 |
| `int16` | 16 бит | −32 768 .. 32 767 |
| `int32` | 32 бит | −2 147 483 648 .. 2 147 483 647 |
| `int64` | 64 бит | −2⁶³ .. 2⁶³−1 |
| `uint8` | 8 бит | 0 .. 255 |
| `uint16` | 16 бит | 0 .. 65 535 |
| `uint32` | 32 бит | 0 .. 4 294 967 295 |
| `uint64` | 64 бит | 0 .. 2⁶⁴−1 |

`int` — псевдоним для `int64`. Используется как тип возврата функции `main`.

### Вещественные типы

| Тип | Размер |
|-----|--------|
| `float32` | 32 бит |
| `float64` | 64 бит |

### Прочие типы

| Тип | Описание |
|-----|----------|
| `bool` | Булевое значение: `true` или `false` |
| `char` | 32-битный Unicode scalar value |
| `string` | Строка в кодировке UTF-8, неизменяема |
| `void` | Отсутствие значения. Допускается только как тип возврата функции |

`char` хранит один допустимый Unicode-код в диапазоне `U+0000..U+10FFFF`,
кроме surrogate-диапазона `U+D800..U+DFFF`. Арифметика над `char`
запрещена; для получения числового кода используется `code(c)`, а для
обратного преобразования — `char_from(n)`.

## Составные типы

### Массивы

```
[N]T
```

Массив фиксированного размера `N` элементов типа `T`.
Размер массива является частью типа: `[5]int32` и `[6]int32` — разные, несовместимые типы.
Размер массива — не менее 1; массивы нулевого размера запрещены.

Пример:
```bina
let arr: [3]float64 = [1.0, 2.0, 3.0];
```

Количество элементов в литерале массива должно точно совпадать с размером типа.

### Структуры

Именованный составной тип с полями. Каждое поле имеет имя и тип.

```bina
struct Point {
    x: float64;
    y: float64;
}
```

Два типа структур совместимы только если это одна и та же структура (номинальная типизация).
Структура `A` с полями `{ x: int32 }` и структура `B` с полями `{ x: int32 }` — разные типы.

---

## Синонимы типов

Синоним типа вводит альтернативное имя для существующего типа.

```bina
type Celsius    = float64;
type Fahrenheit = float64;
```

Переменная типа `Celsius` может быть присвоена переменной типа `Fahrenheit`,
и наоборот, так как это один и тот же тип `float64`.

## Типы литералов

### С явным суффиксом

Если литерал имеет суффикс, его тип определяется суффиксом:

| Литерал | Тип |
|---------|-----|
| `42i8` | `int8` |
| `42i16` | `int16` |
| `42i32` | `int32` |
| `42i64` | `int64` |
| `42u8` | `uint8` |
| `42u16` | `uint16` |
| `42u32` | `uint32` |
| `42u64` | `uint64` |
| `3.14f32` | `float32` |
| `3.14f64` | `float64` |
| `1e6f32` | `float32` |
| `1.5e10f64` | `float64` |

Hex- и binary-литералы имеют те же правила типов, что и десятичные
целочисленные литералы:

| Литерал | Тип |
|---------|-----|
| `0xFFu8` | `uint8` |
| `0b1010i32` | `int32` |

### Без суффикса (вывод типа по умолчанию)

| Литерал | Тип по умолчанию |
|---------|------------------|
| `42` | `int32` |
| `0xFF` | `int32` |
| `0b1010` | `int32` |
| `3.14` | `float64` |
| `1e6` | `float64` |
| `inf`, `nan`, `NaN` | `float64` |
| `'a'`, `'\n'`, `'\u{1F600}'` | `char` |
| `"hello"` | `string` |
| `true`, `false` | `bool` |

`inf`, `nan` и `NaN` являются специальными вещественными литералами.
Если рядом есть ожидаемый тип `float32` или `float64`, литерал получает этот
ожидаемый тип. Если ожидаемого float-типа нет, используется `float64`.
`nan` и `NaN` эквивалентны. Значение `-inf` получается как унарный минус
от литерала `inf`.

Ожидаемый тип влияет на литерал только тогда, когда литерал стоит
непосредственно в позиции с известным типом: инициализация переменной с
аннотацией типа, правая часть присваивания, выражение `return`, элемент
литерала массива и значение поля литерала структуры при известном типе,
операнд унарного минуса. В остальных случаях — в том числе в операндах
бинарных операций и в аргументах вызова функции — тип литерала выводится
без учёта внешнего контекста: в `let x: float32 = 1.0 + 2.0;` операнды
имеют тип `float64`, и инициализация является ошибкой компиляции.

## Вывод типов

При объявлении переменной указание типа необязательно.
Если тип опущен, компилятор выводит его из выражения-инициализатора.

```bina
let x = 42;          // int32
let y = 3.14;        // float64
let s = "hello";     // string
let f = true;        // bool
let b = 255u8;       // uint8
```

Правила вывода:

| Инициализатор | Выведенный тип |
|---------------|----------------|
| Целочисленный литерал без суффикса | `int32` |
| Целочисленный литерал с суффиксом | тип суффикса |
| Вещественный литерал без суффикса | `float64` |
| Вещественный литерал с суффиксом | тип суффикса |
| `inf`, `nan`, `NaN` | `float64` или ожидаемый float-тип |
| Символьный литерал | `char` |
| Строковый литерал | `string` |
| Булевый литерал | `bool` |
| Вызов функции | тип возврата функции |
| Литерал массива | `[N]T`, где `N` — количество элементов, `T` — тип элементов |
| Литерал структуры | тип указанной структуры |
| Переменная | тип переменной |
| Бинарное/унарное выражение | тип результата операции |

Если компилятор не может однозначно определить тип — ошибка компиляции.

~~Вывод типа возвращаемого значения функции не поддерживается —
тип возврата функции всегда указывается явно.~~

## Неявное приведение типов

В арифметических и сравнительных операциях допускается автоматическое расширение числовых типов. Сужение запрещено — требуется явный `cast`.

### Иерархия расширения

```
int8  → int16  → int32  → int64
uint8 → uint16 → uint32 → uint64

uint8  → int16
uint16 → int32
uint32 → int64

int32 → float32
int64 → float64
float32 → float64
```

Расширение типа `A` к типу `B` допускается, если из `A` в `B` ведёт путь по
стрелкам (отношение транзитивно). Например, `uint8 → float32` допустимо через
`uint8 → int16 → int32 → float32`; `uint64` не расширяется ни к одному типу.

Этот же граф используется при разрешении перегрузок: длина кратчайшего пути
служит tie-break для выбора более конкретной перегрузки после сравнения
качества `EXACT` / `PROMOTION` / `CONVERSION` (см. `semantics.md`, раздел
«Перегрузка функций»).

### Правила

1. Если оба операнда имеют одинаковый тип — результат имеет этот тип.
2. Если тип одного операнда расширяется до типа другого — компилятор
   автоматически выполняет расширяющее приведение.
3. Знаковые и беззнаковые типы одного размера не совместимы неявно —
   требуется явный `cast`. Беззнаковый тип расширяется до знакового строго
   большего размера (`uint8 → int16` и далее по иерархии).
4. Особый случай: если один операнд имеет тип `int64`, а другой — `float32`,
   общим типом операции является `float64`.
5. Расширения целых типов к вещественным могут терять точность (значение
   округляется до ближайшего представимого), но не диапазон.
6. Строки, `bool`, `char` и `void` не участвуют в неявном приведении.

### Примеры

```bina
let a: int32 = 10;
let b: float64 = 3.14;
let c = a + b;             // a неявно расширяется до float64, результат: float64

let x: int8 = 100;
let y: int32 = x + 1;      // x неявно расширяется до int32

let z: int8 = x + 1;       // результат int32 не может быть сужен до int8
let z: int8 = cast<int8>(x + 1);  // явное приведение
```

### Приведение при присваивании

При присваивании и инициализации неявное расширение также допускается:

```bina
let big: int64 = 42i32;    // int32 → int64 (расширение)
let small: int8 = 42i32;   // int32 → int8 (сужение, требуется cast)
```

## Явное приведение типов

Оператор `cast<T>(expr)` выполняет явное преобразование значения к типу `T`.

Допустимые преобразования:

| Из | В | Поведение |
|----|---|-----------|
| Целое | Целое | Обрезание старших бит при сужении |
| Целое | Вещественное | Преобразование с возможной потерей точности |
| Вещественное | Целое | Отбрасывание дробной части (truncation); вне диапазона целевого типа, для `NaN` и `±inf` — ошибка времени выполнения |
| Вещественное | Вещественное | Округление при сужении |
| `bool` | Целое | `false` → 0, `true` → 1 |
| Целое | `bool` | 0 → `false`, ≠ 0 → `true` |
| Число | `string` | Текстовое представление числа |
| `string` | Целое | Парсинг строки как целого числа |
| `string` | Вещественное | Парсинг строки как вещественного числа |

Недопустимые преобразования (ошибка компиляции):

- `string` ↔ `bool`
- `string` ↔ `char`
- `char` ↔ числовой тип через `cast`; для этого используются `code` и `char_from`
- Массив ↔ что-либо
- Структура ↔ что-либо

Строка при приведении к числу должна точно соответствовать формату
числового литерала с необязательным ведущим знаком `-`: для целых —
десятичные цифры, для вещественных — также дробная часть и экспонента.
Пробелы, знак `+`, шестнадцатеричная запись недопустимы. Если строка не
соответствует формату, во время выполнения возникает ошибка:

```
runtime error: invalid numeric conversion at line <N>
```

Если строковое значение выходит за диапазон целевого целочисленного типа,
возникает `runtime error: integer overflow at line <N>`.
Для приведения строки к вещественному типу допустимы специальные значения
`"inf"`, `"-inf"`, `"nan"` и `"NaN"`.

Если при приведении вещественного значения к целому типу результат после
отбрасывания дробной части не помещается в целевой тип, а также для значений
`NaN` и `±inf`, возникает `runtime error: integer overflow at line <N>`.

---

## Поведение при переполнении (Overflow)

Переполнение целочисленных типов — ошибка во время выполнения.
При возникновении переполнения программа завершается с сообщением:

```
runtime error: integer overflow at line <N>
```

Переполнение вещественных типов приводит к значениям `±Inf` или `NaN`.
