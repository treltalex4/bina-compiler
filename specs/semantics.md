# Bina Language Semantics

## Стратегия вычисления

Язык Bina использует **call-by-value** — все аргументы передаются в функцию
по значению. При вызове функции каждый аргумент копируется в локальную
переменную-параметр. Изменение параметра внутри функции не влияет на значение
в вызывающем контексте.

```bina
fn increment(x: int32) -> int32 {
    return x + 1;
    // x — локальная копия, оригинал не затронут
}
```

---

## Семантика присваивания

Все присваивания выполняют копирование по значению (value semantics).

```bina
let a: [3]int32 = [1, 2, 3];
mut b: [3]int32 = a;   // b — независимая копия
b[0] = 999;            // a[0] остаётся равным 1
```

Это относится ко всем типам: числам, строкам, массивам и структурам.

---

## Инициализация переменных

Каждая переменная должна быть инициализирована в момент объявления.
Использование неинициализированной переменной — ошибка компиляции.

```bina
let x: int32 = 0;      // ок
let y: int32;           // ошибка компиляции: отсутствует инициализатор
```

---

## Мутабельность

Переменные объявленные через `let` — иммутабельные (неизменяемые).
Переменные объявленные через `mut` — мутабельные (изменяемые).

```bina
let x: int32 = 10;
x = 20;                // ошибка компиляции: x иммутабельна

mut y: int32 = 10;
y = 20;                // ок
```

Поля структур наследуют мутабельность переменной, в которой хранится структура:

```bina
mut p: Point = Point { x: 1.0, y: 2.0 };
p.x = 3.0;             // p мутабельна

let q: Point = Point { x: 1.0, y: 2.0 };
q.x = 3.0;             // ошибка: q иммутабельна
```

## Время жизни переменных

Переменная существует в пределах блока `{ ... }`, в котором объявлена.
По выходу из блока переменная уничтожается.

```bina
fn example() -> void {
    let x: int32 = 10;     // x существует здесь
    if true {
        let y: int32 = 20; // y существует только внутри if
    }
    // y больше не существует
    // x всё ещё существует
}
// x больше не существует
```

## Область видимости и разрешение имён

### Лексическая область видимости

Идентификатор разрешается в ближайшей охватывающей области видимости,
начиная с текущего блока и поднимаясь вверх. Для переменных поиск ограничивается
телом функции (глобальных переменных в языке нет), а для функций и типов —
поднимается до пространств имён и глобального уровня.

### Теневое перекрытие (Shadowing)

Внутреннее объявление имени скрывает внешнее в пределах своего блока:

```bina
let x: int32 = 10;
if true {
    let x: int32 = 20;  // новый x, скрывает внешний
    print(x);            // выводит 20
}
print(x);                // выводит 10 (внешний x не изменён)
```

### Запрет использования до объявления

Идентификатор не может быть использован до его объявления в текущей области:

```bina
print(x);               // ошибка: x ещё не объявлен
let x: int32 = 42;
```

### Запрет повторного объявления

Одно и то же имя не может быть объявлено дважды в одной области видимости:

```bina
let x: int32 = 1;
let x: int32 = 2;       // ошибка: x уже объявлен в этой области
```

## Пространства имён

Пространство имён группирует объявления под общим именем.
Доступ к элементам — через оператор `::`.

```bina
namespace Math {
    fn abs(x: int32) -> int32 {
        if x < 0 {
            return 0 - x;
        }
        return x;
    }
}

fn main() -> int {
    let result = Math::abs(0 - 5);
    print(result);       // 5
    return 0;
}
```

Пространства имён могут быть вложенными:

```bina
namespace A {
    namespace B {
        fn f() -> void { print("A::B::f"); }
    }
}

fn main() -> int {
    A::B::f();
    return 0;
}
```

## Методы структур (impl)

Блок `impl` привязывает функции (методы) к конкретной структуре.

### Правила

1. Первый параметр метода должен называться `self` и иметь тип
   соответствующей структуры. Параметр `self` передаётся по значению (копия).
2. При вызове метода через точку (`obj.method(args)`) объект автоматически
   подставляется как первый аргумент (`self`).
3. Один блок `impl` на одну структуру. Повторный `impl` для той же структуры —
   ошибка компиляции.
4. Блок `impl` должен располагаться в той же области видимости,
   что и объявление структуры.

### Пример

```bina
struct Circle {
    radius: float64;
}

impl Circle {
    fn area(self: Circle) -> float64 {
        return 3.14159 * self.radius * self.radius;
    }

    fn scale(self: Circle, factor: float64) -> Circle {
        return Circle { radius: self.radius * factor };
    }
}

fn main() -> int {
    let c = Circle { radius: 5.0 };
    let a = c.area();           // эквивалентно Circle::area(c)
    print(a);

    let big = c.scale(2.0);    // эквивалентно Circle::scale(c, 2.0)
    print(big.area());
    return 0;
}
```

### Вызов как обычной функции

Методы также можно вызвать как обычные функции через квалифицированное имя:

```bina
let c = Circle { radius: 5.0 };
let a = Circle::area(c);       // явный вызов, эквивалентно c.area()
```

## Семантика выражений

### Арифметические операции

| Оператор | Операнды | Результат |
|----------|----------|-----------|
| `+` | два числа | число общего расширенного типа |
| `+` | две строки | конкатенация (новая строка) |
| `-` | два числа | число общего расширенного типа |
| `*` | два числа | число общего расширенного типа |
| `/` | два числа | число общего расширенного типа |
| `%` | два целых числа | целое число общего расширенного типа |

Деление целых чисел — целочисленное (отбрасывание дробной части).
Деление на ноль — ошибка во время выполнения.

Если типы операндов различаются, применяются правила неявного приведения
(см. `types.md`, раздел «Неявное приведение типов»). Результат будет иметь 
общий расширенный тип.

### Операции сравнения

| Оператор | Операнды | Результат |
|----------|----------|-----------|
| `==`, `!=` | два значения | `bool` |
| `<`, `>`, `<=`, `>=` | два числа | `bool` |

Строки сравниваются лексикографически (побайтово, UTF-8).
Сравнение `==` и `!=` допускается для строк.
Сравнение `<`, `>`, `<=`, `>=` для строк **не допускается**.

Цепочки сравнений (`a < b < c`) запрещены — сравнение не ассоциативно.

### Логические операции

| Оператор | Описание | Вычисление |
|----------|----------|------------|
| `&&` | логическое И | ленивое (short-circuit) |
| `\|\|` | логическое ИЛИ | ленивое (short-circuit) |
| `!` | логическое НЕ | — |

Операнды должны иметь тип `bool`. При ленивом вычислении:
- `a && b` — если `a` равно `false`, `b` не вычисляется.
- `a || b` — если `a` равно `true`, `b` не вычисляется.

### Унарные операции

| Оператор | Операнд | Результат |
|----------|---------|-----------|
| `-` (унарный минус) | число | число того же типа |
| `!` (отрицание) | `bool` | `bool` |

### Индексирование массива

```bina
let arr: [5]int32 = [10, 20, 30, 40, 50];
let x = arr[2];         // x = 30
```

Индекс должен быть целочисленного типа. Индексация с нуля.
Выход за границы массива — ошибка во время выполнения.

### Доступ к полю структуры

```bina
let p = Point { x: 1.0, y: 2.0 };
let a = p.x;            // a = 1.0
```

Обращение к несуществующему полю — ошибка компиляции.

## Семантика инструкций

### Ветвление (if / else)

Условие должно иметь тип `bool`. Допускается цепочка `else if`:

```bina
if x > 0 {
    print("positive");
} else if x == 0 {
    print("zero");
} else {
    print("negative");
}
```

### Цикл (while)

Условие должно иметь тип `bool`. Тело выполняется, пока условие истинно.

```bina
mut i: int32 = 0;
while i < 10 {
    print(i);
    i = i + 1;
}
```

### break / continue

`break` — немедленный выход из ближайшего цикла `while`.
`continue` — переход к следующей итерации ближайшего цикла `while`.

Использование `break` или `continue` вне тела цикла — ошибка компиляции.

### return

Возврат значения из функции. Тип возвращаемого значения должен совпадать
с типом возврата функции (с учётом неявного расширения).

В функциях с типом возврата `void` — `return;` без значения.

Если управление достигает конца тела функции с не-`void` типом возврата
без выполнения `return` — ошибка компиляции.

## Семантика функций

### Объявление

Функция объявляется на верхнем уровне программы, внутри пространства имён
или внутри блока `impl`. Вложенные функции (функция внутри функции) запрещены.

### Точка входа

Программа обязана содержать функцию `main` с типом возврата `int`:

```bina
fn main() -> int {
    return 0;
}
```

Возвращаемое значение `main` передаётся как код завершения процесса.

### Порядок вычисления аргументов

Аргументы функции вычисляются слева направо перед вызовом.

## Встроенные функции

| Функция | Аргументы | Возврат | Описание |
|---------|-----------|---------|----------|
| `print(expr)` | любой тип | `void` | Вывод значения в stdout. Числа и строки выводятся в текстовом виде, `bool` — как `true`/`false`. Добавляет перевод строки. |
| `input()` | нет | `string` | Чтение строки из stdin (до символа новой строки). |
| `len(expr)` | `string` или массив | `int64` | Длина строки (в байтах) или количество элементов массива. |
| `exit(expr)` | целое число | не возвращает | Немедленное завершение программы с указанным кодом возврата. |
| `panic(expr)` | `string` | не возвращает | Аварийное завершение программы. Выводит сообщение в stderr и завершает процесс с кодом 1. |

## Строковые операции

Строки в Bina — неизменяемые (immutable) последовательности байтов в UTF-8.

| Операция | Синтаксис | Описание |
|----------|-----------|----------|
| Конкатенация | `s1 + s2` | Создаёт новую строку |
| Длина | `len(s)` | Количество байтов |
| Сравнение | `s1 == s2`, `s1 != s2` | Побайтовое сравнение |

```bina
let greeting = "hello" + " " + "world";
let length = len(greeting);    // 11
```

## Ошибки во время выполнения

При возникновении ошибки во время выполнения программа завершается
с сообщением в stderr в формате:

```
runtime error: <сообщение> at line <N>
```

Обязательно обрабатываемые ошибки:

| Ошибка | Сообщение |
|--------|-----------|
| Деление на ноль | `division by zero` |
| Выход за границы массива | `index out of bounds` |
| Целочисленное переполнение | `integer overflow` |
